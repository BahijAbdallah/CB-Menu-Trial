Goal
Add an Admin feature to control the DISPLAY ORDER of items inside each category on the frontend. 
Do NOT change item data, images, or any UI design; only the order in which items appear.

Requirements
1) Storage (non-destructive)
- Create a single settings key to store order per category:
  settings.itemOrderByCategory = {
    "<categoryIdOrSlug>": ["itemId1","itemId7","itemId3", ...],
    ...
  }
- Persist via the existing settings/config store (or a minimal app_settings table with key:string PK, value:json).
- Never mutate item records; only read their ids/titles for context.
- Provide Reset (delete the order entry for that category) to return to default order.

2) Admin UI (no design change to site)
- New admin screen: “Category ➝ Item Order”.
- Dropdown/select to choose a category.
- Below it, show a read-only list of that category’s items (title + small thumb) as a **drag-and-drop** list.
- Actions: Save Order, Reset to Default.
- Do not edit item content/images; list is for ordering only.

3) Frontend apply (sorting only, keep current components)
- When rendering a category’s items, sort by the saved order if present; items not listed keep their original order after the listed ones.
- Stable sort; do NOT modify cards, HTML, CSS, or images.

Use this exact, non-invasive sort helper:
--------------------------------------------------
// items: original array for a category (keep as-is)
// categoryId: current category identifier
// settings: object that includes itemOrderByCategory map

function sortItemsForCategory(items, categoryId, settings) {
  const order = settings?.itemOrderByCategory?.[categoryId] || [];
  const pos = new Map(order.map((id, i) => [id, i]));
  // preserve original order for items not in 'order'
  return items
    .map((it, idx) => ({ it, idx }))
    .sort((a, b) => {
      const ai = pos.has(a.it.id) ? pos.get(a.it.id) : Number.MAX_SAFE_INTEGER;
      const bi = pos.has(b.it.id) ? pos.get(b.it.id) : Number.MAX_SAFE_INTEGER;
      if (ai !== bi) return ai - bi;
      return a.idx - b.idx; // stable fallback
    })
    .map(x => x.it);
}
--------------------------------------------------

4) API (if needed)
- GET /api/settings/item-order → returns itemOrderByCategory map
- POST /api/settings/item-order  body: { categoryId, order: ["itemId", ...] } → upsert only this category’s array
- DELETE /api/settings/item-order?categoryId=... → remove that entry to reset

5) Acceptance
- Admin can reorder items within any category and Save; frontend reflects the new order immediately on reload.
- Reset returns to previous default (no custom order).
- No item content/images updated; no UI design changes anywhere.
- Existing pagination, filters, and search continue to work with the new order.
